# Codex Working Rules

Bearbeite niemals komplette Dateien,
wenn eine gezielte Änderung genügt.

Immer zuerst prüfen,
ob bestehende Komponenten erweitert werden können.

Nach jeder Änderung

npm run build

Keine projektspezifischen Begriffe im Core.

Content niemals mit Icons versehen.

Renderer übernimmt Darstellung.

Neue Komponenten müssen Multi-Projekt-fähig sein.

Bei Architekturfragen immer PROJECT_RULES.md beachten.
