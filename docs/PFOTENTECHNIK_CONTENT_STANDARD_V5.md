# PFOTENTECHNIK_CONTENT_STANDARD_V5

> Redaktionshandbuch für alle Inhalte auf PfotenTechnik.de

# Leitprinzip

Jeder Artikel muss die Suchintention **vollständig**, **präzise** und
**besser als die Top-10-Ergebnisse** beantworten. SEO ist das Ergebnis
hochwertiger Inhalte, nicht deren Ausgangspunkt.

------------------------------------------------------------------------

# 1. Artikeltypen

## Wissensratgeber

Ziel: Symptome, Ursachen, Vorgehen, Prävention.

Pflicht: - Premium Answer - Quick Facts - Entscheidungsbaum -
Information-Gain - FAQ - Quellen

## Vergleichsseite

Ziel: Produkte vergleichen.

Pflicht: - Vergleichstabelle - Testergebnis - Kaufberatung - FAQ -
Alternativen - Fazit

## Einzelprodukt

Ziel: Produkt verstehen und einordnen.

Pflicht: - Kurzbewertung - Vor-/Nachteile - Für wen geeignet -
Alternativen - FAQ

------------------------------------------------------------------------

# 2. Suchintention zuerst

Vor dem Schreiben festlegen:

-   Informational
-   Commercial Investigation
-   Transactional
-   Navigational

Die Struktur richtet sich danach.

------------------------------------------------------------------------

# 3. Qualitätsmodell

Jeder Artikel beantwortet:

1.  Was ist das Problem?
2.  Warum entsteht es?
3.  Wie erkennt man es?
4.  Was kann man selbst tun?
5.  Wann wird es gefährlich?
6.  Wann zum Tierarzt?
7.  Wie lässt es sich verhindern?

------------------------------------------------------------------------

# 4. Information Gain

Mindestens fünf unterschiedliche Elemente.

Geeignet:

-   Mythos vs Realität
-   Häufige Denkfehler
-   Fallbeispiel
-   Zeitachse
-   Prozentrechner
-   Wasser-/Energieberechnung
-   Entscheidungsmatrix
-   Ursachen nach Wahrscheinlichkeit
-   Tierarztperspektive
-   typische Diagnostik
-   Verlauf
-   Unterschiede nach Alter
-   typische Fehlentscheidungen
-   Messmethoden

Nicht zwei Artikel mit identischer Auswahl erstellen.

------------------------------------------------------------------------

# 5. Redundanzen vermeiden

Nicht:

-   dieselbe Aussage mehrfach
-   FAQ wiederholt Überschriften
-   mehrere Checklisten mit gleichem Inhalt

Ein Gedanke erscheint genau einmal.

------------------------------------------------------------------------

# 6. Struktur

1.  Frontmatter
2.  Hero
3.  Premium Answer
4.  Quick Facts
5.  Inhaltsnavigation
6.  Hauptartikel
7.  Information Gain
8.  Tabellen
9.  Entscheidungsbaum
10. Tierarzt-Checkliste
11. Abschlusscheckliste
12. FAQ
13. Quellen

------------------------------------------------------------------------

# 7. Tabellen

2--4 Tabellen.

Keine Tabelle ohne Erkenntnisgewinn.

------------------------------------------------------------------------

# 8. FAQ

12--18 Fragen.

Antworten:

-   konkret
-   keine Wiederholung
-   Longtail-orientiert
-   50--120 Wörter

Schema.org ausschließlich daraus erzeugen.

------------------------------------------------------------------------

# 9. Bilder

Alle Bilder müssen:

-   einen Mehrwert liefern
-   im Fließtext erklärt werden
-   mobil lesbar sein
-   exakt zum Kapitel passen

Keine dekorativen Bilder.

------------------------------------------------------------------------

# 10. Interne Links

Pro Artikel:

-   2--6 Links
-   Hub-Seite
-   verwandte Wissensseite
-   ggf. Vergleichsseite
-   ggf. Produktseite

Anchor natürlich formulieren.

------------------------------------------------------------------------

# 11. E-E-A-T

-   medizinische Aussagen mit Quellen
-   Unsicherheit transparent benennen
-   keine Diagnosen behaupten
-   Notfälle klar markieren
-   seriöse Quellen bevorzugen

------------------------------------------------------------------------

# 12. AI-Overview-Optimierung

Jeder Artikel enthält:

-   direkte Antwort innerhalb der ersten 200 Wörter
-   Entscheidungsbaum
-   mindestens zwei strukturierte Tabellen
-   Definitionen wichtiger Begriffe
-   konkrete Zahlen nur mit Quelle
-   Zusammenfassungen nach langen Abschnitten

------------------------------------------------------------------------

# 13. Stil

-   kurze Absätze
-   aktive Sprache
-   präzise Formulierungen
-   keine Buzzwords
-   keine KI-Floskeln
-   kein Keyword-Stuffing

------------------------------------------------------------------------

# 14. Technische SEO

-   eine H1
-   logische H2/H3
-   sprechende Alt-Texte
-   Bildgrößen definiert
-   CLS vermeiden
-   Hero priorisiert laden
-   übrige Bilder lazy

------------------------------------------------------------------------

# 15. Visual-Strategie

Jeder Wissensartikel nutzt möglichst:

-   Hero
-   Entscheidungsbaum
-   Ursachenübersicht
-   Prozessgrafik
-   Spezialgrafik
-   Tierarztgrafik
-   Abschlusscheckliste

Vorhandene Grafiken werden wiederverwendet.

------------------------------------------------------------------------

# 16. Qualitätscheck (Pflicht)

## Inhalt

-   Suchintention erfüllt
-   Information Gain vorhanden
-   keine Redundanzen
-   logisch aufgebaut

## Medizin

-   Quellen aktuell
-   Warnzeichen korrekt
-   keine unbelegten Aussagen

## SEO

-   interne Links
-   FAQ
-   Tabellen
-   Bilder
-   natürliche Keywords

## Leser

-   mobil gut lesbar
-   klare Handlungsempfehlungen
-   verständliche Sprache

------------------------------------------------------------------------

# 17. Freigabekriterien

Ein Artikel ist erst fertig, wenn:

-   alle Pflichtmodule vorhanden sind
-   FAQ nur einmal existiert
-   maximal zwei Checklisten verwendet werden
-   Information-Gain-Elemente abwechslungsreich sind
-   keine Aussage doppelt vorkommt
-   alle Bilder korrekt referenziert sind
-   Markdown valide ist
-   Rechtschreibung geprüft wurde
-   Quellen vollständig sind

Dieser Standard ist verbindlich für alle zukünftigen Inhalte von
PfotenTechnik.de.
