# PFOTENTECHNIK_EDITORIAL_STANDARD_2026

> Version 2026.1 Stand: 2026-07-17

Verbindliches Redaktionshandbuch für PfotenTechnik.de.

## 1. Redaktionelle Philosophie

Pfotentechnik ist ein unabhängiges Fachportal für smarte
Haustiertechnik. Jeder Inhalt muss nachvollziehbar, transparent und
hilfreicher als bestehende Suchergebnisse sein.

## 2. Qualitätsprinzipien

-   Helpful Content First
-   EEAT vor SEO
-   Information Gain
-   Mobile First
-   Transparenz
-   Keine unbelegten Aussagen

## 3. Content-Typen

Cornerstone, Wissensartikel, Vergleich, Produktseite und Herstellerseite
besitzen jeweils definierte Ziele, Pflichtmodule und interne
Verlinkungen.

## 4. Suchintention

Vor jedem Artikel wird die Suchintention dokumentiert und die
Seitenstruktur darauf abgestimmt.

## 5. Informationsarchitektur

Eine H1, logische H2/H3, Premium Answer am Anfang, klare Nutzerführung.

## 6. Mobile First

Maximal zwei Fließtextabsätze, regelmäßige visuelle Unterbrechungen,
kurze Listen.

## 7. Cornerstone

Jedes Kernthema erhält genau eine Hub-Seite. Alle Cluster verlinken
zurück.

## 8. Premium Blocks

Premium Answer, Quick Facts, Decision, Warning, Mythos, FAQ, Vergleich,
Glossar.

## 9. Information Gain

Entscheidungsbaum, Matrix, Wartungsplan, Datenschutz, Technik erklärt,
Praxisszenarien.

## 10. Visual Storytelling

Jede größere H2 wird nach Möglichkeit durch Grafik, Tabelle oder
Infografik unterstützt.

## 11. Visual Pack

Zu jedem Wissensartikel gehört eine slug.visual-pack.md mit Dateinamen,
Alt-Text, Position und Prompt.

## 12. SEO

Semantik, interne Links, strukturierte Daten, AI-Overview-Eignung,
Featured Snippets.

## 13. EEAT

Quellen offenlegen, Bewertungsmethodik erklären, Grenzen benennen,
Aktualität sicherstellen.

## 14. Schreibstil

Aktive Sprache, kurze Sätze, keine KI-Floskeln, keine Redundanzen.

## 15. Bildrichtlinien

Nur informative Grafiken, keine Logos, keine Wasserzeichen, Alt-Texte
verpflichtend.

## 16. Qualitätskontrolle

Check auf Lesbarkeit, Quellen, FAQ, Bilder, Links, Frontmatter,
Markdown.

## 17. Freigabekriterien

Premium Ready nur bei vollständigem Visual Pack, EEAT und bestandenem
Review.

## 18. Autoren-Checkliste

-   [ ] Suchintention festgelegt

-   [ ] Premium Answer vorhanden

-   [ ] Information Gain integriert

-   [ ] Visual Pack erstellt

-   [ ] Interne Links gesetzt

-   [ ] Quellen geprüft

-   [ ] FAQ einzigartig

-   [ ] Alt-Texte vorhanden

-   [ ] Mobile geprüft

-   [ ] Keine Redundanzen

-   [ ] Markdown valide

-   [ ] Abschlussreview erfolgt

## 19. Verbote

-   Keyword-Stuffing

-   Erfundene Testergebnisse

-   Doppelte FAQ

-   Dekorative Bilder ohne Mehrwert

-   Überlange Absätze

-   Unbelegte Gesundheitsversprechen

-   Irreführende Überschriften
